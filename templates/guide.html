使用说明

有效期

有效期决定分享在服务器上保留多久。到期后，内容会在下一次清理或访问时被移除。

{% if args.enable_burn_after %}

阅后销毁

阅后销毁会限制可读取次数。达到次数后，分享会被删除，适合一次性或临时内容。

{%- endif %} {% if args.highlightsyntax %}

语法高亮

如果分享的是代码，可以选择自动识别或指定语言，查看时会更容易阅读。

{%- endif %}

分享码

分享码是可选的自定义短名称，只能使用 3-64 位英文字母、数字、短横线或下划线。设置后可以直接访问 {{ args.public_path_as_str() }}/分享码，例如分享码为 1234 时，打开 {{ args.public_path_as_str() }}/1234 就能查看对应内容。

如果不填写分享码，系统仍会自动生成随机码。系统路径如 list、upload、raw 等不能作为分享码。

密码

密码用于保护私密分享、公开但需要密码的分享，以及只读分享的修改/删除操作。系统不会把密码保存在“我的分享”里。

可见性

可见性决定别人能否在列表里看到分享，以及查看时是否需要密码。

公开

公开内容会显示在列表里，任何访问者都可以打开。适合公告、公开资料或无敏感信息的内容。

仅链接可见

不显示在公开列表里，但知道链接或分享码的人可以访问。

公开但需要密码

会显示在列表里，方便接收方发现；但打开正文、原始内容、附件或跳转时都需要输入密码。

只读

知道链接的人可以查看，但修改或删除需要密码。

私密

不显示在列表里，知道链接后仍需要输入密码才能查看。

本地加密

内容在浏览器端先加密，再发送到服务器。安全性更高，但访问时需要在浏览器里解密。

我的分享

“我的分享”只保存在当前浏览器的本地存储里，用来帮你找回自己刚创建过的链接。它不会同步到服务器，也不会保存密码。失效链接会自动从列表页隐藏。
